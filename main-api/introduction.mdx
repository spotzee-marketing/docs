The Main API is the primary way to integrate Spotzee with your own systems. Use it to sync contacts, trigger campaigns, build segments, manage journeys, and administer projects and organisations.

## Base URL

```text
https://apix.spotzee.com/api
```

The public surface is split across two prefixes:

| Prefix | Key prefix | Use |
|---|---|---|
| `/client/*` | `sk_` (secret), `pk_` (publishable) | Third-party integrations, agent workloads, browser/mobile clients (publishable on an explicit allowlist) |
| `/admin/*` | `ok_` (organisation) | Cross-project administration |

## Authentication

All requests require a bearer token. See the [Authentication](/guides/authentication) guide for key types, scopes, and rotation.

```http
Authorization: Bearer YOUR_API_KEY
```

## Conventions

Common request and response conventions live in the Guides tab:

| Concern | Read |
|---|---|
| Versioning | [API versioning](/guides/api-versioning) — `Spotzee-Version` header |
| Rate limits | [Rate limits](/guides/api-rate-limits) — `X-RateLimit-*` headers, MCP doubling |
| Idempotency | [Idempotency](/guides/api-idempotency) — `Idempotency-Key` on `POST` and `PATCH` |
| Pagination | [Pagination](/guides/api-pagination) — cursor-based with `nextCursor` / `prevCursor` |
| Webhooks | [Webhooks](/guides/api-webhooks) — outgoing signing and retries |
| Correlation | Every response includes `X-Request-Id` — quote it when reporting issues |
