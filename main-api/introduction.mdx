The Main API is the primary way to integrate Spotzee with your own systems. Use it to sync users, trigger campaigns, build segments, manage journeys, and administer projects and organisations.

## Base URL

```text
https://apix.spotzee.com/api
```

The public surface is split across two prefixes:

| Prefix | Key prefix | Use |
|---|---|---|
| `/client/*` | `sk_` (secret), `pk_` (publishable) | Third-party integrations, agent workloads, browser/mobile clients (publishable on an explicit allowlist) |
| `/admin/*` | `ok_` (organisation) | Cross-project administration |

## Authentication

All requests require a bearer token. See the [Authentication](/guides/authentication) guide for key types, scopes, and rotation.

```http
Authorization: Bearer YOUR_API_KEY
```

## Conventions

The Main API is rolling out a set of wire-level conventions in stages. Per-IP edge rate limits, cursor pagination, and the `X-Request-Id` correlation header are live today. Per-key rate limits with read/write split, the MCP boost, `Idempotency-Key` enforcement, full `Spotzee-Version` validation, and the RFC 7807 error envelope ship in a future release. Each guide flags the live-vs-rolling-out state for its concern.

| Concern | Read |
|---|---|
| Versioning | [API versioning](/guides/api-versioning): `Spotzee-Version` header |
| Rate limits | [Rate limits](/guides/api-rate-limits): per-IP edge limits today, per-key budgets rolling out |
| Idempotency | [Idempotency](/guides/api-idempotency): `Idempotency-Key` rolling out on Main API |
| Pagination | [Pagination](/guides/api-pagination): cursor-based with `nextCursor` / `prevCursor` |
| Webhooks | [Webhooks](/guides/api-webhooks): subscriptions configurable today; outbound delivery on the roadmap |
| Correlation | Every response includes `X-Request-Id`. Quote it when reporting issues. |
