The **Main API** is the primary way to integrate Spotzee with your own systems. Use it to sync contacts, trigger campaigns, build segments, manage journeys, and administer projects and organisations.

## Base URL

```
https://apix.spotzee.com/api
```

## Authentication

All requests require a bearer token. See [Authentication](/guides/authentication).

```http
Authorization: Bearer YOUR_API_KEY
```

## Surfaces

The Main API exposes two surfaces, distinguished by URL prefix and key type:

| Prefix | Key prefix | Use |
|---|---|---|
| `/client/*` | `sk_` (secret), `pk_` (publishable) | Third-party integrations, agent workloads, browser/mobile clients (publishable only on an explicit allowlist) |
| `/admin/*` | `ok_` (organisation) | Cross-project administration: create projects, mint project keys |

See [Authentication](/main-api/authentication) for how to choose between key types.

## Conventions

| Concern | Convention |
|---|---|
| Versioning | `Spotzee-Version` header, date-based — see [Versioning](/main-api/versioning) |
| Rate limits | Per-key budgets, `X-RateLimit-*` response headers — see [Rate limits](/main-api/rate-limits) |
| Idempotency | `Idempotency-Key` header on `POST` and `PATCH` — see [Idempotency](/main-api/idempotency) |
| Pagination | Cursor-based with `nextCursor` / `prevCursor` — see [Pagination](/main-api/pagination) |
| Errors | RFC 7807 `application/problem+json` — see [Errors](/main-api/errors) |
| Correlation | Every response includes an `X-Request-Id` header — quote it when reporting issues |
