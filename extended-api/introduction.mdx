The **Extended API** is a set of low-latency edge utilities that complement the Main API. Use it for high-volume operations where round-trip latency matters: email address validation, DNS lookups for domain setup, SMS number validation, and AI subject-line scoring.

## Base URL

```
https://apix.spotzee.com/api/ext
```

Both the Main API and Extended API now serve from the same hostname. The Extended API ships the wire-level conventions live today (`Spotzee-Version`, `X-Request-Id`, RFC 7807 error envelope). The Main API is rolling those conventions out in stages. See [API conventions](/guides/api-versioning) for the full set.

## Authentication

All requests require a bearer token. The same API keys used for the Main API are accepted here. See [Authentication](/guides/authentication).

```http
Authorization: Bearer YOUR_API_KEY
```

## Capabilities

| Group | What it does |
|---|---|
| Email | Address validation, syntax + DNS sanity checks, MX/SPF/DMARC lookup, domain blacklist + IP blacklist checks, SenderScore lookup, malicious URL lookup, SMTP server + send tests, disposable-email detection, mailbox-provider identification, one-shot tester inboxes |
| DNS | DoH lookups (A, AAAA, MX, TXT, CNAME, NS, SOA, PTR, SRV, CAA + DNSSEC), TCP and URL connectivity probes, end-to-end domain health check |
| SMS | E.164 phone-number validation and ISO country resolution |
| AI | Subject-line variant generation and 14-criterion subject-line scoring |
| Text | Readability scores (Flesch + 6 indices, reading time) and sentiment analysis |
| Data | Generate synthetic person, contact, name, and date records |

## When to use Extended over Main

| Use Extended API for… | Use Main API for… |
|---|---|
| Per-request validation in a signup form | CRUD on users, segments, journeys |
| Domain-setup wizards in your own UI | Triggering campaigns and journeys |
| High-volume scoring at the edge | Reading reporting and analytics data |
| Quota and credit lookups | Managing organisation and project state |

## Versioning

The Extended API is dated like the Main API. Pin a version with the `Spotzee-Version` request header (see [API versioning](/guides/api-versioning)). The current release is `2026-04-28`.

The OpenAPI specification is published at:

```
https://apix.spotzee.com/api/ext/openapi.json
```

## Rate limits and quota

Two separate gates apply to Extended API traffic.

**Edge rate limit.** Spotzee's edge network enforces a per-source-IP cap of 1,000 requests per 10 seconds across `apix.spotzee.com/api/*`. Over-limit traffic receives `429 Too Many Requests` with the canonical error envelope. The cap counts every request, authenticated or not.

**Credit-based quota.** Validation, scoring, lookup, and generation endpoints deduct credits on success. The cost varies by route. Inspect remaining balance via the `Quota` endpoints before high-volume runs.

Per-API-key sliding-window rate limits and a doubled budget for MCP traffic are rolling out alongside the Main API hardening rollout. The Extended API does not honour those today.
