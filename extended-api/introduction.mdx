The **Extended API** is a set of low-latency edge utilities that complement the Main API. Use it for high-volume operations where round-trip latency matters: email address validation, DNS lookups for domain setup, SMS dispatch, and AI text scoring.

## Base URL

```
https://apix.spotzee.com/api/ext
```

Both the Main API and Extended API now serve from the same hostname. The same `Spotzee-Version`, `X-Request-Id`, and error envelope conventions apply on both surfaces — see [API conventions](/guides/api-versioning) for the full set.

## Authentication

All requests require a bearer token. The same API keys used for the Main API are accepted here. See [Authentication](/guides/authentication).

```http
Authorization: Bearer YOUR_API_KEY
```

## Capabilities

| Group | What it does |
|---|---|
| Email health | Validate addresses, detect role accounts, check domain reputation |
| DNS | Look up SPF, DKIM, DMARC, MX records — used by domain-setup wizards |
| SMS | Dispatch SMS messages and check delivery |
| AI text tools | Score, classify, and generate content snippets |
| Customer data | Read-only lookups for enriched customer records |
| Quota | Inspect remaining credits on metered operations |

## When to use Extended over Main

| Use Extended API for… | Use Main API for… |
|---|---|
| Per-request validation in a signup form | CRUD on users, segments, journeys |
| Domain-setup wizards in your own UI | Triggering campaigns and journeys |
| High-volume scoring at the edge | Reading reporting and analytics data |
| Quota and credit lookups | Managing organisation and project state |

## Versioning

The Extended API is dated like the Main API. Pin a version with the `Spotzee-Version` request header (see [API versioning](/guides/api-versioning)). The current release is `2026-04-28`.

The OpenAPI specification is published at:

```
https://apix.spotzee.com/api/ext/openapi.json
```

## Rate limits

Per-API-key rate limits apply. Quota-metered endpoints (validation, scoring) deduct credits on success — check `Quota` endpoints before high-volume runs.
