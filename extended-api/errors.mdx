The Extended API and Main API share one error envelope, served as `application/problem+json` (RFC 7807). Every non-2xx response carries the same fields.

## Error body shape

```json
{
  "status": "error",
  "code": "authentication_required",
  "title": "Authentication required",
  "message": "Invalid API key. Please ensure you copied it correctly.",
  "error": "Invalid API key. Please ensure you copied it correctly.",
  "type": "https://docs.spotzee.com/main-api/errors#authentication_required",
  "request_id": "req_9f3bad665eabf0d6"
}
```

| Field | Type | Description |
|---|---|---|
| `status` | string | Always `"error"` for non-2xx responses. |
| `code` | string | Stable, machine-readable identifier. Safe to switch on; never localised. |
| `title` | string | Short human-readable summary suitable for a heading. en-AU. |
| `message` | string | Human-readable explanation. May reference identifiers. |
| `error` | string | Legacy mirror of `message`. Retained while existing integrations migrate; new code should read `message`. |
| `type` | string | Documentation URL for this error class (RFC 7807). |
| `request_id` | string | Server-generated correlation ID. Echoed in the `X-Request-Id` response header. Quote it in support tickets. |
| `param` | string | Optional. Names the offending parameter when a single field is at fault. |
| `errors[]` | array | Optional. Field-level breakdown when multiple parameters fail validation. Each entry: `{ param, code, message }`. |

## Status codes

| Status | Meaning | Typical cause |
|---|---|---|
| `400 Bad Request` | Invalid input | Malformed email, invalid domain, bad payload, unsupported `Spotzee-Version` |
| `401 Unauthorized` | Missing or invalid token | No `Authorization` header, expired key |
| `403 Forbidden` | Authenticated but not allowed | Key scope insufficient |
| `404 Not Found` | Resource or record not present | Domain has no DNS record of the requested type |
| `408 Request Timeout` | Upstream lookup exceeded timeout | DNS or upstream provider slow — retry once |
| `429 Too Many Requests` | Rate limited or quota exhausted | Honour `Retry-After`; check your remaining quota |
| `500 Internal Server Error` | Unexpected platform error | Retry with backoff |
| `503 Service Unavailable` | Temporary capacity issue | Retry with backoff |

## Common error codes

The full code catalogue is shared with the Main API:

| `code` | When it appears |
|---|---|
| `authentication_required` | No credential or invalid credential. |
| `permission_denied` | Credential lacks scope or role. |
| `parameter_missing` / `parameter_invalid_format` | Field absent or malformed (e.g. invalid email, bad domain). |
| `validation_failed` | One or more `errors[]` entries present. |
| `quota_exceeded` | Credit-metered operation; insufficient balance. |
| `rate_limited` | Token bucket exhausted; check `Retry-After`. |
| `version_unsupported` / `version_required` | `Spotzee-Version` header value not a known release. |
| `internal_error` | Unhandled — quote `request_id` in support tickets. |

## Quota errors

Validation and AI-scoring endpoints are credit-metered. A `quota_exceeded` response indicates the daily or plan-level credit limit has been reached. Inspect remaining credits via the `Quota` endpoints before high-volume runs.

## Retry guidance

- **Lookups** (`GET`): retry on `5xx`, `408`, and `429` with exponential backoff.
- **SMS dispatch** (`POST`): include an `Idempotency-Key` to make retries safe. Do not retry `4xx`.
