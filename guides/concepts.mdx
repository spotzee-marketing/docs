This page introduces the objects you'll work with throughout the product and the API.

## Organisation

The top-level container. An organisation owns billing, team members, projects, and audit history. Most customers operate one organisation; agencies and groups operate several.

## Project

A workspace inside an organisation. A project owns its own users, segments, journeys, campaigns, templates, and providers. Use separate projects to isolate brands, regions, or production-vs-staging workloads.

## User

A person or account that can be marketed to. Every user has standard identifiers (email, phone) plus arbitrary custom attributes you define. Users can sit in many segments at once.

## Segment

A dynamic audience defined by a query (attributes + behaviour). Membership recalculates as data changes — you do not maintain segment membership manually.

## List

A static, named collection of users. Unlike a segment, a list's membership is set explicitly. Use lists for one-off imports and historical exports.

## Journey

A visual multi-step automation. Spotzee journeys support eleven step types — `send`, `delay`, `gate` (conditional split), `experiment` (A/B), `balancer`, `update`, `event`, `exit`, `link`, `wait-for-event`, and triggers (event-based or scheduled).

## Channel

A delivery surface. Spotzee supports email, SMS, WhatsApp, push, in-app, and webhooks. Each channel uses a configured **provider** to do the actual delivery.

## Provider

The vendor account that actually sends messages. On Starter, you bring your own (any supported email or SMS provider). Professional and Enterprise tiers include managed delivery infrastructure as well as BYO support.

## Template

A reusable message body — for any channel — with merge tags that resolve from user attributes and event data at send time.

## Campaign

A one-off send to a segment or list. Campaigns are simpler than journeys when you only need a single broadcast.

## Event

A behaviour signal sent into Spotzee — a page view, a purchase, a custom action. Events feed segments, trigger journeys, and personalise content.
