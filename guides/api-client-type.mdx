The `x-spotzee-client-type` header tells Spotzee what kind of client made a request. The header is optional, opaque, and reserved for two things.

1. **Rate-limit policy.** Once the per-key sliding-window rate limits ship (see [Rate limits](/guides/api-rate-limits)), MCP and agent traffic gets a doubled budget so parallel tool calls don't run out of headroom mid-task.
2. **Telemetry.** Spotzee uses the value to break traffic down by client type internally, which informs SDK roadmaps and capacity planning.

```http
x-spotzee-client-type: mcp
```

<Note>
  `x-spotzee-client-type` is accepted on every request today (no error if absent or unknown), but the rate-limit policy that would honour it is rolling out behind a feature flag. Including the header now is forward-compatible. Once enforcement ships, the documented values take effect automatically.
</Note>

## Recognised values

Send one of these. Anything else is bucketed as `unknown`.

| Value | Use for |
|---|---|
| `mcp` | MCP servers and agent runtimes that fan out parallel tool calls |
| `mobile` | Native iOS or Android apps using the official mobile SDKs |
| `sdk-js` | The official JavaScript SDK (set automatically) |
| `sdk-android` | The official Android SDK (set automatically) |
| `sdk-ios` | The official iOS SDK (set automatically) |
| `cli` | Command-line tools (deploy scripts, one-off backfills, your own CLI) |
| `unknown` | Default when omitted. No future rate-limit boost. |

## When to set it

<Tabs>
  <Tab title="MCP server">
    Always set `x-spotzee-client-type: mcp`. Once the per-key rate limits ship, this is the only way an agent driving many parallel calls keeps up with its own fan-out.
  </Tab>
  <Tab title="CLI tool">
    Set `x-spotzee-client-type: cli` so Spotzee can distinguish ad-hoc CLI traffic from production application traffic in telemetry.
  </Tab>
  <Tab title="Custom integration (your own backend)">
    Leave the header off. Your traffic is treated as `unknown` and gets the standard rate-limit policy for your account. Setting `mcp` from a non-MCP backend just to grab the future boost is unsupported.
  </Tab>
  <Tab title="Official SDK">
    Don't set it manually. The SDK does this for you. Overriding the value in your own wrapper interferes with Spotzee's ability to attribute traffic correctly.
  </Tab>
</Tabs>
